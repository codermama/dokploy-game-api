# 打地鼠游戏后端 API — PRD

## 项目概述

`dokploy-game-api` 是卡通风格打地鼠小游戏的后端服务，提供分数管理、游戏配置和统计功能。作为独立项目部署到 Dokploy，与前端项目 `dokploy-game-frontend` 通过 REST API + CORS 跨域通信。

## 技术栈

| 组件 | 技术 |
|------|------|
| 框架 | FastAPI (Python 3.12) |
| 数据库 | PostgreSQL (asyncpg)，支持内存模式回退 |
| 容器 | Docker (python:3.12-slim) |
| 端口 | 8000 |

## 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `DATABASE_URL` | PostgreSQL 连接字符串 | 空（内存模式） |

## API 端点

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/health` | GET | 健康检查（含数据库状态） |
| `/api/scores` | GET | 获取排行榜 Top 10 |
| `/api/scores` | POST | 提交分数 `{name, score, level}` |
| `/api/scores/recent` | GET | 获取最近 5 条提交记录大事件 |
| `/api/stats` | GET | 游戏全局统计大盘（总局数/全服最高/均分/玩家数） |
| `/api/game/config` | GET | 游戏全局玩法动态配置参数 |

## 数据模型

### scores 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL | 主键 |
| name | VARCHAR(50) | 玩家名（HTML 转义） |
| score | INTEGER | 分数 (0-99999) |
| level | INTEGER | 等级 (1-100) |
| created_at | TIMESTAMP | 创建时间 |

## 数据验证

- **name**: 1-50 字符，自动 trim + HTML 转义防 XSS
- **score**: 0-99999 范围
- **level**: 1-100 范围

```bash
# Dokploy: 创建 Application → GitHub 仓库 dokploy-game-api
```
